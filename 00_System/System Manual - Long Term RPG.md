# System Manual - Long Term RPG

## Design Idea

You are not managing tasks. You are exploring a knowledge world.

The center of the world is:

[[00 Main Quest - Become a Computational Theoretical Physicist|Become a Computational / Theoretical Physicist]]

Everything else exists to support that main quest.

## World Model

### Main Quest

The core path toward theoretical physics and computational physics:

- Mathematics
- Physics
- English

These are the only permanent active regions.

### Side Quests

Side quests are useful only when they support the main quest.

Rule:

- Active side quests <= 3
- Every active side quest must answer: "How does this help the main quest?"
- Side quests can sleep without guilt.

### Hidden Quests

Hidden quests represent future research maturity:

- Research Taste
- Independent Research
- Scientific Intuition
- Paper Writing
- Advisor Compatibility
- Research Creativity

Early details stay hidden to reduce anxiety and fantasy planning.

## Mastery Tiers

Use tiers instead of percentages.

| Tier | Meaning |
| --- | --- |
| 0 | Heard of it |
| 1 | Basic vocabulary |
| 2 | Basic coursework |
| 3 | Can solve standard problems |
| 4 | Understand formalism |
| 5 | Intuitive understanding |
| 6 | Research-level familiarity |

Tier change should be rare and earned. Daily logs record evidence, not mood.

## Unlock Logic

A node is unlocked when:

- its prerequisites are at the required tier
- it has a real use case
- it supports a current main quest direction

Do not unlock a node just because it is interesting.

## Anti-Overload Rule

If the system feels heavy, delete views, not knowledge.

The minimum viable system is:

- one Canvas
- one dashboard
- one daily log
- one weekly review
- active side quests <= 3
